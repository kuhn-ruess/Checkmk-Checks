PowerShell-based Nagios/MRPE compatible ping check for the Checkmk Windows
agent. Similar in spirit to `check_icmp` but implemented with
`Test-Connection`, so it does not require administrative privileges or a raw
ICMP socket â€” it runs inside the context of the Checkmk agent.

## Package layout

```text
windows_ping/
  src/
    info
    info.json
    agents/custom/windows_ping/check_ping.ps1
  windows_ping-1.0.0.mkp   (generated by ./mkp_packer pack windows_ping)
```

The script is packaged under `agents/custom/windows_ping/` so that it can be
shipped through the Checkmk Agent Bakery rule **"Deploy custom files with
agent rules"**. On the Checkmk server, after installing the MKP, the script
lives at:

```text
~/local/share/check_mk/agents/custom/windows_ping/check_ping.ps1
```

## Invoking the check via MRPE

Add an MRPE entry to the Windows agent configuration
(`check_mk.user.yml`), pointing at the deployed path:

```yaml
mrpe:
  config:
    - check: 'Ping_router'
      plugin: 'powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\ProgramData\checkmk\agent\local\custom\windows_ping\check_ping.ps1" router.example.com'
```

Classic `mrpe.cfg` format works the same way. After the next agent run,
rediscover services on the host â€” the check appears as `MRPE Ping_router`.

## Parameters

| Parameter      | Default | Description                              |
|----------------|---------|------------------------------------------|
| `-HostName`    | â€”       | Target hostname or IP (required)         |
| `-Count`       | 4       | Number of echo requests                  |
| `-WarningRta`  | 200     | WARN if average RTA >= value (ms)        |
| `-CriticalRta` | 500     | CRIT if average RTA >= value (ms)        |
| `-WarningPl`   | 40      | WARN if packet loss >= value (%)         |
| `-CriticalPl`  | 80      | CRIT if packet loss >= value (%)         |

## Output / exit codes

Nagios convention: `0=OK`, `1=WARN`, `2=CRIT`, `3=UNKNOWN`. Status line
followed by performance data that Checkmk picks up automatically:

```
PING OK - router.example.com: rta 2.41ms, lost 0% (4/4) | rta=2.41ms;200;500;0; pl=0%;40;80;0;100 rtmin=1.9ms rtmax=3.1ms
```

## Compatibility

- Windows PowerShell 5.1 (uses `ResponseTime`)
- PowerShell 7+ (uses `Latency`)
